Using xml file to define your beans is not a great idea.
Alternate option is to configure beans in Java file


mvnw [options] [<goals>] [<phases>]


@SpringBootConfiguration

  @Configuration - identifies configuration class (.config , signalling initializing beans)
  @ComponentScan - look for beans from this point in project hierrarchy
  @EnableAutoConfiguration - sensible starter
